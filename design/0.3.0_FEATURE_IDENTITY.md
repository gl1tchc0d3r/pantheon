# Phase 3 Feature: Ao's Identity

**Version**: 0.3.0  
**Feature**: Ao's Identity  
**Status**: Planned  
**Created**: 2026-03-20  
**Updated**: 2026-03-21

---

## Overview

Phase 3 gives Ao a defined identity that shapes how it responds. Ao loads two files:
- **SOUL.md** - the unchanging core essence (created once, never modified)
- **IDENTITY.md** - the current behavior and guidelines (editable by user)

Both are injected into every prompt, with Soul first, then Identity.

**This is what makes Ao feel like a persistent entity rather than a generic LLM wrapper.**

## Scope

### In Scope

| Component | Description |
|-----------|-------------|
| **SOUL.md** | `.ao/SOUL.md` - core essence, created once, never modified |
| **IDENTITY.md** | `.ao/IDENTITY.md` - current behavior, editable by user |
| **Soul loader** | Read soul from file, cache permanently |
| **Identity loader** | Read identity from file, cache in memory |
| **Prompt integration** | Soul block first, then Identity block (before history) |
| **Config options** | `identity.soul_path` and `identity.identity_path` |
| **Commands** | `/soul` to view soul, `/identity` to view identity |
| **Default files** | Starter templates created if files don't exist |

### Out of Scope

- In-app identity editing
- Multiple identities / identity switching
- Identity versioning
- Identity templates gallery

### Step 2: Create Identity Module

Create `src/identity.rs` with Soul and Identity structs:

```rust
use chrono::Utc;
use std::fs;
use std::path::Path;
use thiserror::Error;

#[derive(Debug, Error)]
pub enum IdentityError {
    #[error("Failed to read file: {0}")]
    ReadError(#[from] std::io::Error),
    #[error("File not found: {0}")]
    NotFound(PathBuf),
}

// Soul - the unchanging core essence
pub struct Soul {
    content: String,
    path: PathBuf,
    loaded_at: chrono::DateTime<Utc>,
}

impl Soul {
    pub fn load_or_create(path: &Path) -> Result<Self, IdentityError> {
        if !path.exists() {
            let soul = Self::default_soul();
            soul.save()?;
            return Ok(soul);
        }
        
        let content = fs::read_to_string(path)?;
        Ok(Self {
            content,
            path: path.to_path_buf(),
            loaded_at: Utc::now(),
        })
    }
    
    fn default_soul() -> Self {
        Self {
            content: DEFAULT_SOUL.to_string(),
            path: PathBuf::from(".ao/SOUL.md"),
            loaded_at: Utc::now(),
        }
    }
    
    fn save(&self) -> Result<(), IdentityError> {
        fs::write(&self.path, &self.content)?;
        Ok(())
    }
    
    pub fn content(&self) -> &str {
        &self.content
    }
}

// Identity - the current behavior
pub struct Identity {
    content: String,
    path: PathBuf,
    loaded_at: chrono::DateTime<Utc>,
}

impl Identity {
    pub fn load_or_create(path: &Path) -> Result<Self, IdentityError> {
        if !path.exists() {
            let identity = Self::default_identity();
            identity.save()?;
            return Ok(identity);
        }
        
        let content = fs::read_to_string(path)?;
        Ok(Self {
            content,
            path: path.to_path_buf(),
            loaded_at: Utc::now(),
        })
    }
    
    fn default_identity() -> Self {
        Self {
            content: DEFAULT_IDENTITY.to_string(),
            path: PathBuf::from(".ao/IDENTITY.md"),
            loaded_at: Utc::now(),
        }
    }
    
    fn save(&self) -> Result<(), IdentityError> {
        fs::write(&self.path, &self.content)?;
        Ok(())
    }
    
    pub fn content(&self) -> &str {
        &self.content
    }
    
    pub fn reload(&mut self) -> Result<(), IdentityError> {
        self.content = fs::read_to_string(&self.path)?;
        self.loaded_at = Utc::now();
        Ok(())
    }
}

const DEFAULT_SOUL: &str = r#"# Ao's Soul

Your essence is curiosity and a genuine desire to help.

- You think step-by-step through problems

You are not:
```

### Step 4: Update Agent Loop

Modify `src/agent.rs` to include Soul and Identity:

```rust
use crate::identity::{Identity, Soul};
use crate::provider::{LlmProvider, ProviderError};
use crate::session::{capitalize, Message, SessionSummary};

pub struct AgentLoop {
    provider: Box<dyn LlmProvider>,
    soul: Option<Soul>,
    identity: Option<Identity>,
}

impl AgentLoop {
    pub fn new(provider: Box<dyn LlmProvider>) -> Self {
        Self {
            provider,
            soul: None,
            identity: None,
        }
    }
    
    pub fn with_soul(mut self, soul: Soul) -> Self {
        self.soul = Some(soul);
        self
    }
    
    pub fn with_identity(mut self, identity: Identity) -> Self {
        self.identity = Some(identity);
        self
    }

    pub async fn run(
        &self,
        input: &str,
        history: &[&Message],
        previous_summaries: &[SessionSummary],
    ) -> Result<String, AgentError> {
        let prompt = self.build_prompt(input, history, previous_summaries);
        let response = self.provider.complete(&prompt).await?;
        Ok(response)
    }

    fn build_prompt(
        &self,
        input: &str,
        history: &[&Message],
        previous_summaries: &[SessionSummary],
    ) -> String {
        let mut prompt = String::new();

        // Soul block FIRST
        if let Some(soul) = &self.soul {
            prompt.push_str("=== Ao's Soul ===\n");
            prompt.push_str(soul.content());
            prompt.push_str("=== End Soul ===\n\n");
        }

        // Identity block SECOND
        if let Some(identity) = &self.identity {
            prompt.push_str("=== Ao's Identity ===\n");
            prompt.push_str(identity.content());
            prompt.push_str("=== End Identity ===\n\n");
        }

        // Previous session summaries
        if !previous_summaries.is_empty() {
            prompt.push_str("=== Previous Sessions Summary ===\n");
            for summary in previous_summaries {
                prompt.push_str(&format!(
                    "Session {}: {}\n",
                    summary.session_date.format("%Y-%m-%d"),
                    summary.summary
                ));
            }
            prompt.push_str("=== End Previous Sessions ===\n\n");
        }

        // Current session history
        if !history.is_empty() {
            prompt.push_str("=== Current Session History ===\n");
            for msg in history {
                prompt.push_str(&format!("{}: {}\n", capitalize(&msg.role), msg.content));
            }
            prompt.push_str("=== End Current History ===\n\n");
        }

        prompt.push_str(&format!("User: {}\nAo:", input));

        prompt
    }
}
```

**Deliverable**: Soul and Identity blocks prepended to all prompts.

### Step 5: Wire Up in Main

Modify `src/main.rs` to load Soul and Identity at startup:

```rust
use crate::identity::{Identity, Soul, IdentityError};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    tracing_subscriber::fmt::init();
    let config = Config::load()?;
    
    // Load Soul (created once if doesn't exist)
    let soul = match Soul::load_or_create(&config.identity.soul_path.into()) {
        Ok(s) => {
            tracing::info!("Loaded soul from {}", config.identity.soul_path);
            s
        }
        Err(e) => {
            tracing::warn!("Failed to load soul: {}, using embedded default", e);
            Soul::default_soul()
        }
    };
    
    // Load Identity (created if doesn't exist)
    let identity = match Identity::load_or_create(&config.identity.identity_path.into()) {
        Ok(id) => {
            tracing::info!("Loaded identity from {}", config.identity.identity_path);
            id
        }
        Err(e) => {
            tracing::warn!("Failed to load identity: {}, using default", e);
            Identity::default_identity()
        }
    };
    
    let provider = OpenRouterProvider::new(/* ... */);
    let agent = AgentLoop::new(Box::new(provider))
        .with_soul(soul)
        .with_identity(identity);
    
    // ... rest of main loop
}
```

**Deliverable**: Soul and Identity loaded at startup with automatic creation.

```rust
pub enum InputResult {

pub fn handle_input(input: &str) -> InputResult {
    match input.trim() {

        "/help" => InputResult::Command("help".to_string()),
        "/clear" => InputResult::Command("clear".to_string()),

        "/soul" => InputResult::Command("soul".to_string()),
        "/identity" => InputResult::Command("identity".to_string()),

    println!("=== Ao's Identity ===");
```

## Notes

- Soul and Identity are cached in memory, not re-read every prompt
- Soul is permanent—never modified by the system or user
- Identity is intentional—treat as version-controlled config; edit file directly or restore from backup
- No in-app reset command—identity should feel stable, not disposable
- Both files should be version-controllable (just markdown)
- The two-file system aligns with "Ao is the only permanent entity" from architecture
  - Soul = the permanent essence
  - Identity = the current expression of that soul
