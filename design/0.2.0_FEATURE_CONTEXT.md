## Overview

Phase 2 adds conversation history and session persistence to Pantheon. The assistant remembers what you said during the current session, includes previous messages in LLM prompts, and stores all session data in SQLite permanently.

**Cross-Session Continuity**: When a session ends, the LLM generates a brief summary (2-3 paragraphs) of the conversation. On new session start, previous session summaries are fetched from the database and injected into the prompt—giving the assistant context of past conversations within a configurable token budget.

**Token Budget**: The `max_history_tokens` budget applies ONLY to the previous session summaries at session start. During the session, current messages are included without truncation.

**Data Persistence**: All messages are stored PERMANENTLY in the database. Messages are NEVER deleted.

This creates:
- **Within-session continuity**: Assistant references earlier parts of the conversation
- **Cross-session continuity**: Assistant knows what you discussed in previous sessions
- **Token budget control**: `max_history_tokens` limits previous summaries at session start
- **Complete audit trail**: All messages stored permanently for future reference

## Scope

### In Scope

| Component | Description |
|-----------|-------------|
| **Session Store** | SQLite-backed session storage with sessions and messages tables (permanent storage) |
| **Session Manager** | Create/load sessions, track active session, persist to SQLite |
| **History Tracking** | Store all messages (user/ao) with timestamps during session |
| **Prompt Integration** | Include conversation history and session summaries in LLM prompts |
| **Token Budget** | `max_history_tokens` controls context size at session start |
| **Session Summarization** | LLM generates brief summary on session close |
| **Cross-session Summaries** | Previous session summaries injected at session start |
| **Commands** | `/clear` (clear from prompt), `/status` (show session info) |
| **Session Resumption** | Continue previous session on startup (auto-load last session) |

### Out of Scope

These features are deferred to later phases:

- Lineage tracking for compression events
- Session naming and manual session selection
- Full memory system with importance scoring (Phase 5+)
- Frozen memory snapshots (Phase 3+)
- User profile snapshots
- Session history for workers (see below)

### Note: Workers vs Ao

**Session summarization is Ao-only.** Workers are ephemeral agents spawned for specific parallel tasks and do not receive session history or summaries.

| Agent | Receives Session Context? |
|-------|--------------------------|
| **Ao** | Yes - full session history + previous session summaries |
| **Workers** | No - only task-specific context injected by Ao at spawn time |

When Ao spawns a worker, it selects **relevant** context from summaries if needed for the task, but does not inject the full conversation history. Workers are short-lived and task-focused.

See [ARCHITECTURE.md](ARCHITECTURE.md) - "The Worker System" for details on worker prompt injection.

### Data Flow

**On New Session Start:**
```
1. Create new session in memory
          ↓
2. Query previous session summaries from DB (ordered by recency)
          ↓
3. Select summaries that fit within max_history_tokens (oldest first, newest kept)
          ↓
4. Inject into prompt as "Previous Sessions Summary"
          ↓
5. Session ready - no current messages yet
```

**During Session:**
```
1. User types message
          ↓
2. SessionManager stores user message (PERMANENT in DB)
          ↓
3. Agent Loop builds prompt with:
   - Identity (from Phase 3, stub for now)
   - Previous session summaries (set at session start)
   - Current session messages (all accumulated messages)
          ↓
4. LLM processes prompt
          ↓
5. SessionManager stores ao message (PERMANENT in DB)
          ↓
6. TUI displays response
```

**On Session Close:**
```
1. Detect session end (/quit or app exit)
          ↓
2. Assemble full session transcript from stored messages
          ↓
3. Call LLM to generate summary (2-3 paragraphs)
          ↓
4. Store summary in sessions table (messages remain in DB permanently)
```

### Database Schema

```sql
-- Sessions table (updated)
CREATE TABLE IF NOT EXISTS sessions (
    id TEXT PRIMARY KEY,
    name TEXT,
    created_at TEXT NOT NULL,
    last_active_at TEXT NOT NULL,
    summary TEXT,                    -- LLM-generated summary of the session
    summary_tokens INTEGER DEFAULT 0  -- Token count of the summary
);

-- Messages table
CREATE TABLE IF NOT EXISTS messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    role TEXT NOT NULL,
    content TEXT NOT NULL,
    created_at TEXT NOT NULL,
    FOREIGN KEY (session_id) REFERENCES sessions(id)
);

-- Indexes for efficient queries
CREATE INDEX IF NOT EXISTS idx_messages_session ON messages(session_id, created_at);
CREATE INDEX IF NOT EXISTS idx_sessions_last_active ON sessions(last_active_at DESC);
```

### Step 5: Update Agent Loop

Modify `src/agent.rs` to build prompts with history and previous session summaries:

```rust
impl AgentLoop {
    pub async fn run(
        &self,
        input: &str,
        history: &[Message],
        previous_summaries: &[SessionSummary],
    ) -> Result<String, AgentError> {
        let prompt = self.build_prompt(input, history, previous_summaries);
        let response = self.provider.complete(&prompt).await?;
        Ok(response)
    }

    fn build_prompt(
        &self,
        input: &str,
        history: &[Message],
        previous_summaries: &[SessionSummary],
    ) -> String {
        let mut prompt = String::new();
        
        // Identity block (placeholder for Phase 3)
        // prompt.push_str("You are Ao...\n\n");
        
        // Previous session summaries
        if !previous_summaries.is_empty() {
            prompt.push_str("=== Previous Sessions Summary ===\n");
            for summary in previous_summaries {
                prompt.push_str(&format!(
                    "Session {}: {}\n",
                    summary.session_date.format("%Y-%m-%d"),
                    summary.summary
                ));
            }
            prompt.push_str("=== End Previous Sessions ===\n\n");
        }
        
        // Current session history
        if !history.is_empty() {
            prompt.push_str("=== Current Session History ===\n");
            for msg in history {
                prompt.push_str(&format!("{}: {}\n", capitalize(&msg.role), msg.content));
            }
            prompt.push_str("=== End Current History ===\n\n");
        }
        
        // Current input
        prompt.push_str(&format!("User: {}\nAo:", input));
        
        prompt
    }
}

fn capitalize(s: &str) -> String {
    let mut c = s.chars();
    match c.next() {
        None => String::new(),
        Some(f) => f.to_uppercase().collect::<String>() + c.as_str(),
    }
}
```

**Deliverable**: Agent Loop includes previous session summaries and current history in prompts.

```rust
pub struct SessionSummarizer<'a> {
    llm: &'a dyn LlmProvider,
}

impl<'a> SessionSummarizer<'a> {
    pub fn new(llm: &'a dyn LlmProvider) -> Self {
        Self { llm }
    }

    pub async fn summarize(&self, session: &Session) -> Result<String, SummarizeError> {
        // Build transcript
        let transcript = self.build_transcript(session);
        
        // Call LLM to summarize
        let prompt = format!(
            "Summarize the following conversation in 2-3 paragraphs. \
            Focus on: main topics discussed, key decisions made, and any \
            pending or unfinished tasks.\n\n\
            === Conversation Transcript ===\n\
            {}\n\
            === End Transcript ===\n\n\
            Summary:",
            transcript
        );
        
        let summary = self.llm.complete(&prompt).await?;
        Ok(summary.trim().to_string())
    }

    fn build_transcript(&self, session: &Session) -> String {
        session.messages
            .iter()
            .map(|m| format!("{}: {}", capitalize(&m.role), m.content))
            .collect::<Vec<_>>()
            .join("\n")
    }
}
```

**Deliverable**: Session summarization with LLM, outputs 2-3 paragraph summary.

```rust
#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Initialize
    tracing_subscriber::fmt::init();
    let config = Config::load()?;
    let provider = OpenRouterProvider::new(/* ... */);
    let agent = AgentLoop::new(Box::new(provider));
    let mut chat = Chat::new(config).await?;
    
    // Load previous session summaries for cross-session context
    let previous_summaries = chat.session_manager.get_previous_summaries().await?;
    
    // Main loop
    loop {
        let input = read_line();
        
        match chat.handle_input(&input) {
            InputResult::Quit => {
                // Summarize session before exit
                if let Err(e) = chat.end_session(&*provider).await {
                    tracing::warn!("Failed to summarize session: {}", e);
                }
                break;
            }
            InputResult::Command("clear") => {
                chat.clear_history();
                println!("History cleared.");
            }
            InputResult::Command("status") => {
                let session = chat.session_manager.current_session();
                if let Some(s) = session {
                    println!("Session: {}", s.id);
                    println!("Messages: {}", s.messages.len());
                    println!("Tokens: ~{}", s.estimate_tokens());
                    println!("Created: {}", s.created_at);
                } else {
                    println!("No active session.");
                }
            }
            InputResult::Chat(text) => {
                chat.add_user_message(&text)?;
                
                let history = chat.get_history();
                let response = agent.run(&text, &history, &previous_summaries).await?;
                
                chat.add_ao_message(&response)?;
                println!("\nAo: {}", response);
            }
            _ => {}
        }
    }
    
    Ok(())
}
```

**Deliverable**: Main loop with session management, history truncation, and summarization on close.

```rust
// Simple token estimator (words + overhead approximation)
pub fn estimate_tokens(text: &str) -> usize {
    // Rough estimate: ~1.3 tokens per word + 4 tokens per line
    let word_count: usize = text.split_whitespace().count();
    let line_count: usize = text.lines().count();
    (word_count as f32 * 1.3) as usize + (line_count * 4)
}

impl Message {
    pub fn estimate_tokens(&self) -> usize {
        estimate_tokens(&self.content)
    }
}

impl SessionSummary {
    pub fn estimate_tokens(&self) -> usize {
        estimate_tokens(&self.summary)
    }
}
```

| Previous summaries | Send messages, restart | New session shows context of previous |
| `/status` | Type `/status` | Shows session ID, message count, tokens, timestamps |
| `/clear` | Type `/clear` | History cleared, shows "Messages: 0" |
| History in prompt | `What did I say earlier?` | LLM references current session messages |
| Previous context | `What did we discuss yesterday?` | LLM references previous session summary |
| Previous summaries | Have multiple past sessions | Previous session summaries included at session start |
| Session close | Type `/quit` | Summary generated, stored in database |
| App restart | Exit and restart | New session starts with previous summaries injected |

## Open Questions

All questions resolved—see decisions below.

| Question | Decision | Rationale |
|----------|----------|-----------|
| **Session naming** | Defer to Phase 6+ | Adds complexity; ID is sufficient for MVP. Can add `/session name <name>` later. |
| **Manual session selection** | Defer to Phase 6+ | Auto-resume covers 90% of use cases. Session picker adds UI complexity. |
| **History format** | Use `=== Previous Sessions Summary ===` and `=== Current Session History ===` | Clear delimiters help LLM distinguish context layers. |
| **Cross-session context** | LLM-generated summaries on session close | Provides meaningful context without storing full history; token budget controlled via config. |
| **During-session truncation** | Not needed | Token budget only applies to previous summaries at session start; current session messages grow without truncation |

## Notes

- **SQLite chosen** for simplicity and async support via sqlx
- **Messages are permanent**: All messages stored in DB forever.
- **Token budget**: Applies only to previous session summaries at session start. Current session messages are included without truncation.
- **Session summarization** provides cross-session continuity without full history storage
- **Auto-resume by default** provides continuity without explicit user action
- **No lineage tracking** in Phase 2—deferred to Phase 7 when compression events are tracked
- **All open questions resolved**—see Open Questions section for decisions
- **Phase 1 to Phase 2 transition**: See "Current Codebase State" section for details on modifying existing 0.1.0 files
